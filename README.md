# Moa — Money Optimiser Advisor

A holistic, NZ-flavoured financial advisor POC. You describe your money in one form — cash flow, debts, mortgage, KiwiSaver, lump sum, goals — and get back a prioritised plan (financial order of operations) explained in plain English by an LLM, with the option to "what if" the mortgage and download the result as PDF.

The plan engine is deterministic Python. The conversational explanation is generated by a local Ollama model (selectable in the UI). If Ollama is unreachable the backend serves a templated narrative; the demo never breaks.

## What's in a plan

For every profile, the waterfall engine emits an ordered sequence of steps:

1. **Starter emergency fund** — 1 month of expenses.
2. **High-interest debt** — anything > 8% APR, attacked in descending-rate order.
3. **KiwiSaver employer match** — bump employee contribution to 3% if under.
4. **Full emergency fund** — 3 / 4 / 6 months by job stability, +1 per dependent.
5. **Mortgage vs invest** — compares after-tax mortgage rate to after-PIE expected index return, with a 1.5% risk buffer.
6. **KiwiSaver beyond match** — only when PIE-adjusted return beats RWT.
7. **Taxable investing** — leaf split between a 12-month term deposit (safer) and a global index fund (growth), sized to risk preference.

Each step shows its share of the user's monthly surplus and reconciles back to the total. The mortgage step also surfaces the user's current vs new minimum payment, the recommended total, and the fixed-term expiry date.

## Stack

- **Backend** — FastAPI 0.115, Pydantic v2, httpx, pytest + pytest-asyncio + respx, ruff. Python 3.11.
- **Frontend** — React 18 + Vite 5 + Tailwind 3. No router; single page.
- **Infra** — Docker Compose (`docker-compose.yml` dev, `docker-compose.prod.yml` prod). Makefile is the entrypoint.

## Getting started

```bash
cp .env.example dev.env       # local-only, gitignored
make dev                      # http://localhost:5173 (FE), http://localhost:8000 (BE)
make test                     # backend pytest in the dev container
make lint                     # ruff (eslint target tolerated)
make prod                     # detached prod stack
make clean                    # tear both stacks down
make                          # show all targets
```

Backend tests run inside the dev container; the compose `--env-file dev.env` flag is required, so `dev.env` must exist locally.

## Configuration

Everything in `.env.example` (copy to `dev.env`; `prod.env` for prod). Key flags:

### Rates

```
RATE_PROVIDER_TYPE=mock          # or "live" — selects ExternalRatesService backend
API_NINJAS_KEY=                  # required if live
RATES_CACHE_TTL_SECONDS=3600
```

The live provider falls back to mock data on any failure (`source="live-fallback"`).

### Narrative LLM

The UI has a small bar above the intake — pick **Local (Ollama)** or **No LLM (templated)** and an Ollama model from the populated dropdown. Choice persists in `localStorage`. Each `/api/advisor/plan` and `/api/advisor/whatif/mortgage` call threads the choice through `?narrative_provider=…&narrative_model=…`. No restart needed to swap.

```
OLLAMA_BASE_URL=http://host.docker.internal:11434
ADVISOR_NARRATIVE_ENABLED=true
ADVISOR_MAX_OUTPUT_TOKENS=800
```

Ollama models are discovered dynamically via the local server's `/api/tags`. The backend container reaches the host's Ollama via `host.docker.internal` (mapped via `extra_hosts` in compose for Linux).

The cardinal LLM rule: **the LLM never invents numbers**. It only paraphrases the structured plan; numbers are quoted verbatim from the waterfall output.

### Cross-device dev / reverse proxy

```
VITE_API_BASE_URL=               # empty -> client picks at runtime
VITE_ALLOWED_HOSTS=              # comma-separated extra hostnames Vite accepts; "*" disables check
CORS_ORIGINS=http://localhost:5173
```

At runtime the client decides where the backend is:

- On a **non-standard port** (typical dev: `:5173`) → uses `<hostname>:8000` so direct LAN access works (any device on the same network hits `http://<host-ip>:5173`).
- On a **standard port** (no port, 80, 443) — implying a reverse proxy fronts the app — uses a **same-origin relative `/api/...`** path so the proxy can route it.

For the reverse-proxy setup (e.g. nginx-proxy-manager), proxy `/api` → backend `:8000` on the same hostname. CORS already permits any localhost / RFC1918 LAN origin via regex.

## API surface

All under `/api/*`:

```
GET  /api/health
GET  /api/rates                       MarketSnapshot (mock or live with fallback)
POST /api/advisor/plan                FinancialProfile -> {plan, narrative, narrative_source, market_snapshot}
POST /api/advisor/whatif/mortgage     {profile, override} -> same shape
GET  /api/advisor/models?provider=... {provider, available, models[]}
```

`narrative_provider` and `narrative_model` are optional query params on the two POST endpoints.

## Tests

110 backend tests across:

- `test_nz_tax.py` — PAYE bracket edges, ACC cap, KiwiSaver, MTC piecewise, PIE / RWT.
- `test_waterfall.py` — priority monotonicity, lump-sum reconciliation, mortgage-vs-invest crossover (parametrised), step ordering invariants.
- `test_finance.py` — primitive math, sensitivity ordering, taxable leaf.
- `test_advisor_llm.py` — Ollama success/failure paths, fallback contract, model listing.
- `test_advisor_router.py` — endpoint shape, what-if isolation, models endpoint.

All run offline against mocked transports. Frontend has no tests; `make lint` and a Vite build catch the obvious breakage.

## Print / PDF

Click **Download PDF** in the results header. Browser print dialog → Save as PDF. The print stylesheet inverts the dark theme for paper, expands all step cards, and strips the intake form / interactive panels / chrome.

## Demo-never-breaks contract

Both the rates provider and the LLM narrative service catch every exception and degrade to a deterministic fallback (mock rates / templated narrative). The frontend shows an amber ribbon when narrative falls back, but the numbers are identical either way. Do not tighten the broad `except Exception` in `services/rates.py` or `services/advisor_llm.py`.

## What this is not

An educational POC. Not personalised financial advice. Always consult a registered NZ adviser before acting.
